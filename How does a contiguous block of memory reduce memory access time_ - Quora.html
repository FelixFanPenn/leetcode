There are three possible answers depending on size.

Data moves from main memory to cache in blocks of 64 or 128 bytes called a line of cache. So at the small level, once you reference a byte of data in cache then the other bytes of data in the same block are also in cache and can be accessed more quickly. Also, the memory can read the next block of memory into cache faster than it can access any other memory, because once the address has been transmitted to the DRAM the chips are already pointing to the address of the next higher block of memory and do not have to be sent a new address.

At a higher level, when DRAM reads data, it stages a block of 1024 or 2048 (or maybe larger) units of data from the memory transistors to a buffer area that can respond to finer requests (for bytes, words, or a line of cache). The next address that requires a different block to move to the buffer incurs a delay known as the CAS delay, which is usually the first number in the memory timing specifications and is much longer than the delay to access a different data address that is already in the buffer.

At a software level, virtual memory is divided into pages which can be 4K or 64K or 1M in size (depending on CPU and OS). If you access another address in the same page it is known to be in memory, but an address in a different page may require I/O to fetch the data from disk.

So memory is designed to be localized, and it is faster to access an address nearby the previous address than it is to access data farther away.
